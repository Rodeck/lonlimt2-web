## Setup

```bash
# 1. Clone the repository (if not already cloned)
git clone <repo-url> lonlimt2-webpage
cd lonlimt2-webpage

# 2. Switch to the feature branch
git checkout 001-homepage-scaffold

# 3. Install dependencies
npm install

# 4. Build CSS (Tailwind generates public/css/style.css)
npm run build:css
```

- Navbar: LonliMT2 logo | Home (active) | Patch Notes | Presentation | Register/Login

## Deployment (FreeBSD 14)

```bash
# 1. Build CSS for production
npm run build:css

# 2. Start with PM2
npm install -g pm2
pm2 start npm --name lonlimt2 -- start
pm2 save

# 3. Configure PM2 to start on boot
pm2 startup freebsd
# Follow the output instructions to register the startup script
```

**Static file serving**: The Hono app serves files from `public/` directly. No separate
Nginx/Caddy is required for this feature, though a reverse proxy is recommended for
production TLS termination (out of scope here).

## Adding / Modifying Translations

1. Edit `locales/pl.json` (authoritative — all keys must exist here).
2. Add the same key to `locales/en.json` with the English translation.
3. In templates, use `<%= it.t('key.path') %>` (Eta syntax).
4. Restart the dev server (i18n files are loaded once at startup in development).

**Key naming convention**: dot-notation namespaced by page/component:
```
nav.home
nav.patchNotes
hero.tagline
hero.cta
footer.copyright
page.home.title
```
